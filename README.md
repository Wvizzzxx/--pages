# ГрузЭкспресс — Сайт грузоперевозок

Многостраничный сайт услуг грузоперевозок с админ-панелью, системой авторизации и управлением заказами.

## 🚀 Быстрый старт

### Предварительные требования

- **Node.js** >= 18.x
- **npm** >= 9.x
- **MongoDB** (локально или Docker)

### 1. Установка зависимостей

```bash
npm install
```

### 2. Настройка MongoDB

**Вариант A: Локальная MongoDB**

Убедитесь, что MongoDB запущена на `mongodb://localhost:27017`.

**Вариант B: Docker**

```bash
docker run -d -p 27017:27017 --name mongodb mongo:latest
```

### 3. Создание тестовых данных

```bash
cd apps/api
npm run seed
```

Это создаст:
- Администратора: `admin@gruzexpress.ru` / `admin123`
- 4 услуги для демонстрации

### 4. Запуск проекта

**Запуск всех сервисов одновременно:**

```bash
npm run dev
```

**Или запуск отдельных сервисов (в разных терминалах):**

```bash
# Терминал 1: Бэкенд API (порт 4000)
cd apps/api && npm run dev

# Терминал 2: Основной сайт (порт 3000)
cd apps/web && npm run dev

# Терминал 3: Админ-панель (порт 3001)
cd apps/admin && npm run dev
```

### 5. Доступ к приложению

| Сервис | URL | Описание |
|--------|-----|----------|
| **Основной сайт** | http://localhost:3000 | Сайт для клиентов |
| **Админ-панель** | http://localhost:3001 | Панель управления |
| **API** | http://localhost:4000 | REST API |
| **API Health** | http://localhost:4000/api/health | Проверка статуса |

## 🔌 API Endpoints

### Аутентификация
| Метод | Путь | Описание | Доступ |
|-------|------|----------|---------|
| POST | `/api/auth/register` | Регистрация | Публичный |
| POST | `/api/auth/login` | Авторизация (JWT) | Публичный |
| GET | `/api/auth/me` | Текущий пользователь | Авторизованный |

### Услуги
| Метод | Путь | Описание | Доступ |
|-------|------|----------|---------|
| GET | `/api/services` | Список услуг (active) | Публичный |
| GET | `/api/services/:slug` | Детальная информация | Публичный |
| GET | `/api/services/admin/all` | Все услуги | Админ |
| POST | `/api/services` | Создание услуги | Админ |
| PATCH | `/api/services/:id` | Обновление услуги | Админ |
| DELETE | `/api/services/:id` | Удаление услуги | Админ |

### Заказы
| Метод | Путь | Описание | Доступ |
|-------|------|----------|---------|
| POST | `/api/orders/public` | Создание заказа (без авторизации) | Публичный |
| POST | `/api/orders` | Создание заказа | Авторизованный |
| GET | `/api/orders/my` | Мои заказы | Авторизованный |
| GET | `/api/orders/all` | Все заказы | Админ |
| PATCH | `/api/orders/:id/status` | Обновление статуса | Админ |
| DELETE | `/api/orders/:id` | Удаление заказа | Админ |

### Пользователи
| Метод | Путь | Описание | Доступ |
|-------|------|----------|---------|
| GET | `/api/users` | Список пользователей | Админ |
| PATCH | `/api/users/:id/role` | Изменение роли | Админ |
| DELETE | `/api/users/:id` | Удаление пользователя | Админ |

### Загрузка файлов
| Метод | Путь | Описание | Доступ |
|-------|------|----------|---------|

## 🐛 Устранение проблем

### MongoDB не подключается
- Убедитесь, что MongoDB запущена: `mongod --version`
- Или используйте Docker: `docker run -d -p 27017:27017 mongo`
- Проверьте URI в `apps/api/.env`

### Порты заняты
- Остановите процессы, использующие порты 3000, 3001, 4000
- Или измените порты в конфигурационных файлах

### Ошибки при запуске
- Удалите `node_modules` и выполните `npm install` заново
- Проверьте версию Node.js: `node --version` (требуется >= 18)
