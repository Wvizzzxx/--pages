# Инструкция по настройке и запуску ГрузЭкспресс

## Проблема: Форма входа не работает

Если на странице входа (`/login`) нет тестовых данных или нельзя войти, выполните следующие шаги:

---

### 1. Убедитесь, что MongoDB запущена

**Windows:**
```bash
# Проверьте, запущена ли MongoDB
mongosh

# Если не запущена, запустите как сервис:
net start MongoDB
```

**Если MongoDB не установлена:**
- Скачайте: https://www.mongodb.com/try/download/community
- Или используйте Docker: `docker run -d -p 27017:27017 --name mongodb mongo:latest`

---

### 2. Запустите seed для создания тестовых данных

Seed создаёт:
- ✅ Администратора: `admin@gruzexpress.ru` / `admin123`
- ✅ 4 услуги по умолчанию

```bash
cd apps/api
npm run seed
```

```
👤 Создание администратора...
✅ Админ создан: admin@gruzexpress.ru / admin123
📦 Создание услуг...
✅ Создано 4 услуг:
   — Городская доставка
   — Междугородние перевозки
   — Офисные переезды
   — Такелажные работы
🎉 Seed завершён успешно!
```

---

### 3. Запустите все сервисы

**Из корня проекта:**
```bash
npm install        # если ещё не установлены зависимости
npm run dev        # запустит API (порт 4000), Web (порт 3000), Admin (порт 3001)
```

---

### 4. Проверьте доступность сервисов

- **API:** http://localhost:4000/api/health
- **Сайт:** http://localhost:3000
- **Админка:** http://localhost:3001

---

### 5. Войдите на сайт

1. Откройте http://localhost:3000/login
2. Используйте тестовые данные (они отображаются на странице):
   - **Email:** `admin@gruzexpress.ru`
   - **Пароль:** `admin123`
3. Нажмите "Войти"

---

### 6. Если всё ещё не работает

#### Проверьте API:
```bash
curl http://localhost:4000/api/health
# Должен вернуть: {"status":"ok","timestamp":"..."}
```

#### Проверьте логин через API напрямую:
```bash
curl -X POST http://localhost:4000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@gruzexpress.ru","password":"admin123"}'
```

Должен вернуть:
```json
{
  "success": true,
  "data": {
    "token": "...",
    "user": {
      "email": "admin@gruzexpress.ru",
      "role": "admin",
      ...
    }
  }
}
```

#### Проверьте услуги:
```bash
curl http://localhost:4000/api/services
# Должен вернуть список услуг с isActive: true
```

## Частые проблемы и решения

### ❌ "MongoDB connection error"
**Решение:** Запустите MongoDB или проверьте `MONGODB_URI` в `apps/api/.env`

### ❌ "Неверный email или пароль"
**Решение:** Запустите `npm run seed` в папке `apps/api`

### ❌ Услуги не отображаются на сайте
**Решение:** 
1. Убедитесь, что `isActive: true` у услуг
2. Проверьте, что API возвращает `data` (исправлено в `services.ts`)

### ❌ Форма логина пустая / нет полей ввода
**Решение:** 
1. Откройте консоль браузера (F12) и проверьте ошибки
2. Убедитесь, что компонент `BaseInput` загружается из `@repo/ui`
